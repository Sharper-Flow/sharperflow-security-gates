Eliminated npm/npx from the executable and high-value surfaces across the Sharper Flow repos, choosing "don't use npm" as a clean supply-chain lever (npm runs dependency lifecycle scripts by default; bun/pnpm/uv block them). Audit found the footprint was much larger than first scoped (~100+ references, mostly low-value prose with judgment cases that would corrupt under blind sed), so the scope was deliberately narrowed to what actually runs or gets copy-pasted: pokeedge-web package.json + all scripts + AGENTS.md + CONTRIBUTING.md + test-file `Run with:` headers + playwright configs + the `docs/specs/ci-workflows.md` spec (flipped to forbid npm), and pokeedge's two frontend dev scripts. Delivered via PRs #96 (CI-path `npm version` hotfix — a regression I had introduced in the tag-only #93), #97 (web scripts + high-value docs), and #281 (pokeedge scripts); all auto-merge-armed. security-gates + the advance plugin were already npm-clean. Long-tail human-doc prose (TESTING.md tables, runbooks, MAINTAINING_API_DOCS, etc.) was intentionally deferred — docs don't execute and the churn/corruption risk outweighed ~zero security value — and genuinely-legitimate references were preserved untouched: the Dockerfile's `npm install -g bun` (bun bootstrap), historical `npm audit` findings reports, the `npm install -g pnpm` install option, the slop-smells `npm audit` detection rule, and the acp-mux package (separate, deferred). Crucially, this change is hygiene/defense-in-depth, not the primary attack defense — that is tracked separately as `enforceLifecycleScriptBlocking` (enforce script-blocking + frozen installs org-wide).
